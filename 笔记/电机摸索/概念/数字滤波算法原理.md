# 数字滤波算法原理

## 概述

本文档详细解析一阶低通滤波器（First-Order Low-Pass Filter）的原理、实现和应用，这是嵌入式系统中最常用的数字滤波算法之一。

---

## 1. 算法公式

### 基本形式
```c
Output = (NewData * k + (256 - k) * OldData) >> 8
```

### 等价形式
```c
Output = (NewData * k + OldData * (256 - k)) / 256
```

### 递归形式（更直观）
```c
Output = OldData + (NewData - OldData) * k / 256
```

---

## 2. 参数说明

| 参数 | 含义 | 范围 |
|------|------|------|
| `NewData` | 新的采样值（原始ADC读数） | - |
| `OldData` | 上一次的滤波输出值 | - |
| `k` | 滤波系数（决定响应速度） | 0 ~ 256 |
| `>> 8` | 除以256（快速位运算） | - |

### 权重分配
- **新数据权重**: `k / 256`
- **旧数据权重**: `(256 - k) / 256`

---

## 3. 工作原理

### 3.1 加权平均机制

滤波器输出是新旧数据的加权平均值：

```
Output = α × NewData + (1 - α) × OldData

其中 α = k / 256
```

### 3.2 k值的影响

| k值 | 新数据权重 | 滤波效果 | 响应速度 | 应用场景 |
|-----|-----------|---------|---------|---------|
| 1 | 0.4% | 很强 | 很慢 | 稳定直流信号、温度测量 |
| 8 | 3.1% | 强 | 慢 | 缓慢变化信号、湿度测量 |
| 16 | 6.3% | 较强 | 较慢 | 压力传感器、液位测量 |
| 32 | 12.5% | 中等 | 中等 | **通用滤波**、电池电压 |
| 64 | 25% | 轻度 | 较快 | 动态信号、电机电流 |
| 128 | 50% | 很弱 | 快 | 实时性要求高的场合 |
| 256 | 100% | 无 | 立即 | 无滤波（直接输出） |

### 3.3 响应曲线

```
输出幅度
    │
1.0 ├─●───────────────────────────── k=256 (无滤波)
    │  ●
0.8 ├    ●●─────────────────────── k=128
    │      ●●
0.6 ├        ●●●─────────────────── k=64
    │          ●●●
0.4 ├            ●●●●●───────────── k=32
    │              ●●●●●●●
0.2 ├                  ●●●●●●●●●●● k=8
    │                        ●●●●●●
0.0 ├────────────────────────────────●●●─→ 时间
    0    1    2    3    4    5    6    7  (时间常数)
```

## 4. 数学本质

### 4.1 模拟原型

这模拟了 **RC低通滤波器** 的离散化形式：

**连续形式（微分方程）：**
```
τ × (dy/dt) + y = x
```

其中：
- `τ` = R × C（时间常数）
- `x` = 输入信号
- `y` = 输出信号

### 4.2 离散化推导

使用**后向差分法**（Backward Euler）：

```
dy/dt ≈ (y[n] - y[n-1]) / T

τ × (y[n] - y[n-1]) / T + y[n] = x[n]

整理得：
y[n] = (T / (τ + T)) × x[n] + (τ / (τ + T)) × y[n-1]

令 α = T / (τ + T)，则：
y[n] = α × x[n] + (1 - α) × y[n-1]
```

### 4.3 频率响应

**截止频率（-3dB点）：**
```
fc = (k / 256) × fs / (2π)

其中 fs 是采样频率
```

**幅频特性：**
```
│H(f)│ = 1 / √(1 + (f / fc)²)
```

**相频特性：**
```
φ(f) = -arctan(f / fc)
```

## 5. 为什么选择256？

### 5.1 计算效率

```c
// 慢速：除法运算（需要几十个时钟周期）
Output = (NewData * k + OldData * (256 - k)) / 256;

// 快速：位移运算（只需1个时钟周期）
Output = (NewData * k + OldData * (256 - k)) >> 8;
```

**性能对比：**
| 运算 | 时钟周期 | 速度 |
|------|---------|------|
| 除法 `÷ 256` | ~32 | 慢 |
| 右移 `>> 8` | 1 | **快32倍** |

### 5.2 精度适中

- **256级精度** = 8位 = 0.4% 分辨率
- 对于大多数应用足够精确
- 避免浮点运算的开销

### 5.3 易于调整

k值选择直观：
```c
// 常用k值（都是2的幂次或接近）
k = 1;    // 0.4%  - 极强滤波
k = 8;    // 3.1%  - 强滤波
k = 16;   // 6.3%  - 较强滤波
k = 32;   // 12.5% - 中等滤波（推荐）
k = 64;   // 25%   - 轻度滤波
k = 128;  // 50%   - 弱滤波
```

// 下一次采样

## 7. 代码实现

### 7.1 基础实现

```c
// 全局变量
static uint16_t FilteredValue = 0;

// 滤波函数（k=32，中等强度）
uint16_t LowPassFilter(uint16_t NewData)
{
    FilteredValue = (NewData * 32 + FilteredValue * (256 - 32)) >> 8;
    return FilteredValue;
}

// 使用
uint16_t adc_raw = ReadADC();
uint16_t adc_filtered = LowPassFilter(adc_raw);
```

### 7.2 通用实现（可配置k）

```c
// 滤波器结构体
typedef struct {
    uint16_t value;
    uint8_t k;
} LowPassFilter_t;

// 初始化
void LowPassFilter_Init(LowPassFilter_t *filter, uint8_t k)
{
    filter->value = 0;
    filter->k = k;
}

// 滤波函数
uint16_t LowPassFilter_Update(LowPassFilter_t *filter, uint16_t input)
{
    filter->value = (input * filter->k +
                     filter->value * (256 - filter->k)) >> 8;
    return filter->value;
}

// 使用示例
LowPassFilter_t voltage_filter;
LowPassFilter_Init(&voltage_filter, 32);  // k=32

uint16_t voltage = LowPassFilter_Update(&voltage_filter, ReadADC());
```

### 7.3 多通道实现

```c
// 4通道滤波器
typedef struct {
    uint16_t ch1, ch2, ch3, ch4;
} MultiChannelFilter_t;

void MultiChannelFilter_Update(MultiChannelFilter_t *f,
                                uint16_t adc1,
                                uint16_t adc2,
                                uint16_t adc3,
                                uint16_t adc4)
{
    f->ch1 = (adc1 * 32 + f->ch1 * 224) >> 8;
    f->ch2 = (adc2 * 32 + f->ch2 * 224) >> 8;
    f->ch3 = (adc3 * 32 + f->ch3 * 224) >> 8;
    f->ch4 = (adc4 * 32 + f->ch4 * 224) >> 8;
}
```

## 9. 高级技巧

### 9.1 自适应滤波

```c
// 根据信号变化率自动调整k值
uint16_t AdaptiveFilter(uint16_t input)
{
    static uint16_t output = 0;
    static uint16_t last_input = 0;

    uint16_t delta = abs(input - last_input);
    uint8_t k;

    // 变化大时用快速k值，变化小时用慢速k值
    if (delta > 100) {
        k = 128;  // 快速响应
    } else if (delta > 50) {
        k = 64;   // 中等响应
    } else {
        k = 16;   // 强滤波
    }

    output = (input * k + output * (256 - k)) >> 8;
    last_input = input;
    return output;
}
```

### 9.2 启动加速

```c
// 首次采样时直接输出，避免启动延迟
uint16_t FastStartFilter(uint16_t input)
{
    static uint16_t output = 0;
    static uint8_t first = 1;

    if (first) {
        output = input;  // 第一次直接输出
        first = 0;
    } else {
        output = (input * 32 + output * 224) >> 8;
    }
    return output;
}
```

### 9.3 抗饱和处理

```c
// 防止累积误差导致输出饱和
uint16_t AntiWindupFilter(uint16_t input)
{
    static int32_t output = 0;
    int32_t temp;

    temp = (input * 32 + output * 224) >> 8;

    // 限制范围
    if (temp > 4095) temp = 4095;
    if (temp < 0) temp = 0;

    output = (int16_t)temp;
    return (uint16_t)output;
}
```

## 12. 常见问题

### Q1: 如何选择k值？

**经验法则：**
```
k = 256  / (1 + 2π × fc × fs)

其中：
fc = 期望的截止频率
fs = 采样频率
```

**示例：**
- 采样率：10kHz
- 期望截止频率：50Hz
- k ≈ 256 / (1 + 2π × 50 × 0.0001) ≈ 32

### Q2: 滤波效果不好怎么办？

**解决方法：**
1. 调整k值（增大加强，减小减弱）
2. 检查采样率是否足够（≥10倍信号频率）
3. 考虑硬件RC滤波预处理
4. 尝试多级滤波（两级小k值）

### Q3: 响应太慢怎么办？

**优化方案：**
1. 减小k值
2. 使用自适应滤波
3. 提高采样率
4. 改用预测算法（如卡尔曼）

### Q4: 整数运算精度不够？

**改进方法：**
```c
// 使用32位累加器提高精度
uint16_t HighPrecisionFilter(uint16_t input)
{
    static uint32_t accumulator = 0;
    accumulator = (input * 32 + accumulator * 224);  // 不右移
    return (uint16_t)(accumulator >> 8);  // 输出时右移
}
```

## 13. 总结

### 优点
✅ **算法简单** - 只需3次运算
✅ **计算高效** - 位运算替代除法
✅ **资源占用少** - 几个字节RAM
✅ **实时性好** - 单周期延迟
✅ **参数直观** - k值易于理解
✅ **适用性广** - 几乎所有慢变信号

### 缺点
❌ **相位滞后** - 不可避免
❌ **截止频率固定** - 不适合宽动态范围
❌ **非线性失真** - 大信号时有压缩

### 最佳实践
1. **k值选择**：从32开始，根据实际调整
2. **初始化**：首次采样直接赋值避免启动延迟
3. **饱和保护**：限制输出范围防止溢出
4. **多级滤波**：两级小k值优于单级大k值
5. **验证方法**：阶跃响应测试观察效果
