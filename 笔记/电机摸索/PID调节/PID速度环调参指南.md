## 1. PID基础理论

### 1.1 PID三要素作用

| 参数 | 数学表达式 | 物理意义 | 作用效果 |
|------|-----------|---------|---------|
| **比例项 (P)** | Kp × e(t) | 当前误差 | 快速响应，消除当前误差 |
| **积分项 (I)** | Ki × ∫e(t)dt | 历史误差累积 | 消除稳态误差 |
| **微分项 (D)** | Kd × de(t)/dt | 误差变化率 | 预测趋势，减少超调 |

### 1.2 位置式vs增量式PID

#### 位置式PID（本项目采用）
```
输出 = Kp×误差 + Ki×误差积分 + Kd×误差微分
```

**特点**：
- ✅ 输出与绝对位置对应
- ✅ 适合速度环、电流环
- ❌ 需要处理积分饱和

#### 增量式PID
```
输出增量 = Δ(Kp×误差 + Ki×误差积分 + Kd×误差微分)
```

**特点**：
- ✅ 无积分饱和问题
- ✅ 适合位置环
- ❌ 需要累加器

## 2. 位置式PID实现原理

### 2.1 定点数运算（关键优化）

#### 为什么需要定点数？
单片机（ARM Cortex-M0内核）**没有硬件FPU**，浮点运算极慢（约100周期）。

#### 解决方案：Q格式定点数
```c
#define PID_SHIFT  10  // 缩放因子

// 初始化：放大系数
Kp = 0.5 × 2^10 = 512
Ki = 0.02 × 2^10 = 20
Kd = 0.1 × 2^10 = 102

// 计算：缩小结果
Output = (Kp × Error + Ki × Integral + Kd × Derivative) >> 10
```

**效果**：
- 浮点运算：0.5 × 1000 = 500.0 （慢）
- 定点运算：(512 × 1000) >> 10 = 500 （快）

### 2.2 核心算法流程

```c
int32_t PID_Location(PID_typedef *PID, int32_t Reference,
                     int32_t Feedback, int32_t OverLimit)
{
    int32_t New_i, Output, KpLimit;
    bool Int_ok;

    // 1. 计算误差
    PID->Ek = Reference - Feedback;

    // 2. 比例项（独立限幅）
    KpLimit = OverLimit >> 2;  // P项上限 = 总上限的1/4
    PID->KpOutput = (PID->Kp * PID->Ek) >> PID->Shift;
    LIMIT(PID->KpOutput, -KpLimit, KpLimit);

    // 3. 积分项（带抗饱和）
    New_i = PID->Integral + PID->Ek;
    PID->KiOutput = (PID->Ki * New_i) >> PID->Shift;

    // 积分限幅检查
    if (ABS(PID->KiOutput) > OverLimit) {
        Int_ok = false;  // 禁止积分更新
    }

    // 4. 微分项
    PID->KdOutput = (PID->Kd * (PID->Ek1 - PID->Ek)) >> PID->Shift;

    // 5. 总输出
    Output = PID->KpOutput + PID->KiOutput + PID->KdOutput;

    // 6. 输出限幅 & 抗积分饱和
    if (Output > OverLimit) {
        Output = OverLimit;
        if (PID->Ek > 0) Int_ok = false;  // 同符号误差，抑制积分
    }
    else if (Output < PID->OutMin) {
        Output = PID->OutMin;
        if (PID->Ek < 0) Int_ok = false;
    }

    // 7. 更新积分项
    if (Int_ok) {
        PID->Integral = New_i;
    }

    // 8. 保存历史误差
    PID->Ek1 = PID->Ek;

    return Output;
}
```

### 2.3 抗积分饱和机制

#### 问题场景
```
目标转速：5000 RPM
当前转速：1000 RPM
误差：4000 RPM → 积分项持续累积 → 积分饱和
```

#### 解决方案
```c
// 规则：输出饱和时，禁止加深饱和的积分
if (Output > OverLimit && Error > 0) {
    Int_ok = false;  // 不更新积分
}
```

## 3. 速度环PID参数分配原则

### 3.1 参数作用详解

#### P参数（比例增益）
```
典型范围：1.0 ~ 5.0
调节原则：
  - P增大 → 响应快，但超调大
  - P减小 → 稳定，但响应慢
  - P过小 → 稳态误差大
  - P过大 → 系统震荡
```

#### I参数（积分增益）
```
典型范围：0.001 ~ 0.05
调节原则：
  - I增大 → 消除误差快，但超调大
  - I减小 → 稳定，但误差消除慢
  - I过小 → 稳态误差无法消除
  - I过大 → 积分饱和，超调严重
```

#### D参数（微分增益）
```
典型范围：0 ~ 0.5
调节原则：
  - D增大 → 超调小，但噪声敏感
  - D减小 → 抗干扰能力强
  - D过小 → 超调大
  - D过大 → 高频震荡
```

### 3.2 速度环特性

#### 速度环特点
1. **惯性大**：电机+负载惯性大，响应慢
2. **带宽低**：典型带宽10-100Hz
3. **滞后明显**：从转矩变化到转速变化有延迟

#### 对PID的影响
- **P不宜过大**：容易引起机械共振
- **I必需**：消除负载变化引起的稳态误差
- **D谨慎使用**：转速测量噪声大，D会放大噪声

### 3.3 不同转速段的参数需求

| 转速段 | P参数 | I参数 | D参数 | 原因 |
|-------|-------|-------|-------|------|
| **低速 (0-2000)** | 较大 (2-3) | 较小 (0.005-0.01) | 可选 (0.2-0.4) | 惯性影响大，需D抑制震荡 |
| **中速 (2000-5000)** | 适中 (2-2.5) | 适中 (0.01-0.02) | 不推荐 (0) | 平衡响应与稳定 |
| **高速 (5000+)** | 较大 (2.5-3.5) | 较大 (0.02-0.03) | 不推荐 (0) | 反电动势强，需强I克服 |

## 4. 分段PID策略

### 4.1 为什么要分段？

不同转速下，电机特性差异巨大：

```
低速段：
  - 反电动势弱 → 转矩能力强
  - 时间常数大 → 响应慢
  - 需要较强P抑制惯性

高速段：
  - 反电动势强 → 转矩能力弱
  - 时间常数小 → 响应快
  - 需要较强I克服反电动势
```

### 4.2 项目分段配置

```c
// 转速分段：每1000 RPM一段
RPM_Index = MotorRPM / 1000;
if (RPM_Index > 8) RPM_Index = 8;

// G1段：1000-1999 RPM

#define SPID_KI_G1   (int32_t)(0.002f * 1024.0f)
#define SPID_KD_G1   (int32_t)(0.2f * 1024.0f)

// G2段：2000-2999 RPM
#define SPID_KP_G2   (int32_t)(1.5f * 1024.0f)
#define SPID_KI_G2   (int32_t)(0.004f * 1024.0f)
#define SPID_KD_G2   (int32_t)(0.4f * 1024.0f)

// G3段：3000-3999 RPM
#define SPID_KP_G3   (int32_t)(2.0f * 1024.0f)
#define SPID_KI_G3   (int32_t)(0.005f * 1024.0f)
#define SPID_KD_G3   (int32_t)(0.4f * 1024.0f)

// G4段：4000-4999 RPM
#define SPID_KP_G4   (int32_t)(2.5f * 1024.0f)
```

### 4.3 参数变化趋势

```
转速上升 → P增大、I增大、D归零

原因：
1. 高速时反电动势强，需要更强I克服
2. 高速时测量噪声大，D会放大噪声
3. 低速时惯性影响大，需要D抑制超调
```

### 4.4 参数切换平滑处理

```c
void sPID_Parament_Updata(PID_typedef *PID, uint16_t MotorRPM)
{
    int32_t CoeffBase, CoeffNext, CoeffPrecent;
    uint16_t RPM_Index;

    RPM_Index = MotorRPM / 1000;
    if (RPM_Index > 8) RPM_Index = 8;

    CoeffBase = (int32_t)(&SPID_Kp_G1 + RPM_Index * 3);  // 基础值
    CoeffNext = CoeffBase + 3;                            // 下一段值

    // 计算插值比例（0-100%）
    CoeffPrecent = MotorRPM % 1000;

    // 线性插值，避免突变
    PID->Kp = CoeffBase + (CoeffNext - CoeffBase) * CoeffPrecent / 1000;
    // ... Ki、Kd同理
}
```

### 5.2 经典Ziegler-Nichols法

#### 步骤1：确定临界增益
```
1. 设置 Ki=0, Kd=0
2. 从小到大增加Kp
3. 记录系统开始震荡的Kp值 = Ku（临界增益）
4. 测量震荡周期 = Tu（临界周期）
```

#### 步骤2：计算初始参数
```
Kp = 0.6 × Ku
Ki = 1.2 × Ku / Tu
Kd = 0.075 × Ku × Tu
```

### 5.3 工程经验法（推荐）

#### 阶段1：纯P调节
```
目标：找到临界P值

步骤：
1. 设置 Ki=0, Kd=0
2. Kp从0.5开始，每次增加0.5
3. 观察转速响应曲线

现象判断：
  - 响应慢，无超调  → Kp太小
  - 轻微超调，快速收敛 → Kp合适 ✓
  - 严重超调，持续震荡 → Kp太大

典型值：Kp = 2.0 ~ 3.0
```

#### 阶段2：加入I消除误差
```
目标：消除稳态误差

步骤：
1. 固定Kp为阶段1值
2. Ki从0.001开始，每次增加0.002
3. 观察稳态误差

现象判断：
  - 稳态误差存在  → Ki太小
  - 误差消除，轻微超调 → Ki合适 ✓
  - 严重超调，积分饱和 → Ki太大

典型值：Ki = 0.01 ~ 0.03
```

#### 阶段3：微调D（可选）
```
目标：减少超调

步骤：
1. 仅在低速段使用D
2. Kd从0.1开始，每次增加0.1
3. 观察超调量

现象判断：
  - 超调明显  → 需要增加D
  - 超调<5%，稳定 → D合适 ✓
  - 高频噪声，抖动  → D太大

典型值：Kd = 0.2 ~ 0.4
```

## 6. 常见问题与解决方案

### 6.1 问题诊断表

| 现象 | 可能原因 | 解决方案 |
|-----|---------|---------|
| **响应慢，无超调** | P太小 | 增大Kp（×1.2） |
| **严重超调** | P太大 | 减小Kp（×0.8） |
| **稳态误差大** | I太小 | 增大Ki（×1.5） |
| **积分饱和** | I太大 | 减小Ki（×0.7）或增强抗饱和 |
| **持续震荡** | P过大或D不当 | 减小P或去掉D |
| **高频抖动** | D过大或噪声 | 减小D或加强滤波 |
| **低速不稳定** | 缺少D项 | 加入适量D（0.2-0.4） |
| **高速跟踪差** | I太小 | 增大Ki以克服反电动势 |

### 6.2 典型问题案例

#### 案例1：低速启动震荡
```
现象：1000 RPM时持续震荡
原因：惯性大，缺少D项
解决：
  Kp = 2.0, Ki = 0.005, Kd = 0.0  → 震荡
  Kp = 2.0, Ki = 0.005, Kd = 0.3  → 稳定 ✓
```

#### 案例2：高速跟踪慢
```
现象：8000 RPM时跟踪慢，误差大
原因：反电动势强，I不足
解决：
  Kp = 3.0, Ki = 0.01, Kd = 0.0  → 误差大
  Kp = 3.0, Ki = 0.025, Kd = 0.0  → 跟踪良好 ✓
```

#### 案例3：负载突变后转速跌落
```
现象：加载后转速恢复慢
原因：I参数太小
解决：
  Ki = 0.01  → 恢复时间2s
  Ki = 0.02  → 恢复时间0.5s ✓
```
